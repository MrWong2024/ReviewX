# ReviewX 后端 Service 职责地图

## 1. 用途

- 记录 Service 职责边界、依赖关系、外部副作用和不应承担的职责
- 防止 Service 职责膨胀或跨模块串台

## 2. 当前状态

- `backend` 已初始化
- 当前已确认最小根服务
- 当前已新增 users 基础 Service
- 当前已新增 sessions 基础 Service
- 当前已新增 auth 第一阶段 Service 和 SessionAuthGuard
